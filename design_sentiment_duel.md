# 设计规范：市场情绪“多空拔河”投票模块 (Sentiment Duel)

## 1. 概述
一个极简的交互式投票组件，旨在收集并展示过去 24 小时内用户对市场的态度（看涨 vs 看跌），通过视觉上的“拔河”效果产生情绪共鸣，提升站点活跃度。

## 2. 功能逻辑
- **核心交互**：二选一投票（看涨 🐂 / 看跌 🐻）。
- **计算机制**：**严格 24 小时滚动窗口**。系统仅统计当前时间向前推 24 小时的有效票数。
- **限制机制**：
    - **Cookie 校验**：本地存储 `last_vote_time`。
    - **后端 IP Hash 校验**：防止清理 Cookie 后恶意刷票。
    - **限制频率**：每台设备 24 小时内仅限投 1 票。

## 3. 技术实现 (Backend)
- **数据库模型**：`PollVote`
    - `id`: Integer, PK
    - `vote_type`: String(10) ['bull', 'bear']
    - `ip_hash`: String(64)
    - `created_at`: DateTime
- **API 接口**：
    - `GET /api/poll/status`: 返回 `{ "bull_pct": 60, "bear_pct": 40, "has_voted": false }`。
    - `POST /api/poll/vote`: 接收 `{ "type": "bull" }`，执行防刷检查并入库。

## 4. 视觉与交互 (Frontend)
- **组件名称**：`SentimentDuel.vue`
- **状态切换**：
    - **未投状态**：展示两个彩色大按钮，背景微动（呼吸效果）。
    - **已投状态**：展示圆角“拔河进度条”，比例变化带弹性动画效果。
- **色彩规范**：
    - 看涨 (Bull)：`#10b981` (Emerald 500)
    - 看跌 (Bear)：`#ef4444` (Red 500)
