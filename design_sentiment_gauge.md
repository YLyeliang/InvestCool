# 设计规范：InvestCool 市场情绪仪表盘

## 1. 核心目标
创建一个半圆型（Speedometer）仪表盘，实时反映当前 NASDAQ 100 市场的综合风险与机会水平。

## 2. 计算模型 (加权 0-100 分)
得分越高代表市场越“贪婪/过热”，得分越低代表市场越“恐惧/低估”。

| 因子 | 权重 | 取值逻辑 (示例) |
| :--- | :--- | :--- |
| **价格位置 (Price Action)** | 35% | RSI(14) > 70 趋向高分，RSI < 30 趋向低分 |
| **波动率 (Volatility)** | 35% | VIX 处于 52 周低位趋向高分，VIX 飙升趋向低分 |
| **估值水位 (Valuation)** | 15% | NDX P/E 处于历史 10 年百分位 |
| **宏观流动性 (Macro)** | 15% | 10 年期美债收益率 (^TNX) 的周变动率 |

## 3. 视觉规范 (半圆型)
- **色段分布**:
  - 0-30: 深绿色 (极度恐惧 - 机会)
  - 30-50: 浅绿色 (恐惧 - 建议定投)
  - 50-70: 黄色 (中性 - 观望)
  - 70-85: 橙色 (贪婪 - 风险)
  - 85-100: 亮红色 (极度贪婪 - 泡沫)
- **技术实现**: Apache ECharts `gauge` 组件。

## 4. 数据流设计
1. **后端 (Flask)**: 每 10 分钟调用 `yfinance` 获取数据并执行加权计算。
2. **存储**: 将最新得分及因子组成存入 SQLite `market_metrics` 表。
3. **API**: 暴露 `GET /api/market-index` 接口。
4. **前端 (Nuxt 4)**: 仪表盘组件定时（10min）轮询该接口。

## 5. 决策记录 (Decision Log)
- **视觉选择**: 半圆型 (简洁、适合仪表盘磁贴布局)。
- **存储方案**: 后端计算+数据库持久化 (性能优、支持后续历史数据回溯)。
- **刷新频率**: 10 分钟 (平衡数据实时性与 API 限制)。
